Componentes en Angular

Los componentes son los bloques fundamentales de cualquier aplicación Angular. Cada componente es una combinación de un template (HTML), una clase (TS) y unos estilos (CSS). Estos permiten que la aplicación sea modular y fácil de mantener.

→ ¿Qué son los Componentes?

En Angular, los componentes son la unidad básica de la interfaz de usuario. Cada componente define una vista que se muestra al usuario. Cada componente consta de tres partes principales:

  • ▪️Clase (TypeScript): Define la lógica del componente, es donde se manipulan los datos y se definen las funciones.
  • ▪️Template (HTML): Define la estructura visual del componente, es decir, lo que el usuario ve.
  • ▪️Estilos (CSS): Define los estilos visuales de la vista (opcional, pero recomendado para mejorar la apariencia).

Estructura de un Componente

La estructura básica de un componente en Angular consiste en tres archivos principales:

  • componente.component.ts: Archivo TypeScript que contiene la lógica del componente.
  • componente.component.html: Archivo HTML que contiene la vista o la estructura visual.
  • componente.component.css: Archivo CSS que define los estilos del componente.

Un componente básico puede verse así:


import { Component } from '@angular/core';

@Component({
  selector: 'app-mi-componente',
  templateUrl: './mi-componente.component.html',
  styleUrls: './mi-componente.component.css'
})
export class MiComponenteComponent {
  titulo: string = 'Hola Mundo';

  constructor() { }
}

1. La Clase (TS)

La clase del componente contiene toda la lógica y los datos necesarios para su funcionamiento. En Angular, la clase se define utilizando el decorador @Component, que se encarga de asociar la clase con su template y estilos. 


import { Component } from '@angular/core';

@Component({
selector: 'app-mi-componente',
templateUrl: './mi-componente.component.html',
styleUrls: './mi-componente.component.css'})
export class MiComponenteComponent {
mensaje: string = 'Este es un componente de ejemplo';

mostrarMensaje() {
console.log({ {this.mensaje}});
  }
}

2. El Template (HTML)

El template de un componente es el archivo HTML donde se define la estructura de la interfaz de usuario. El template puede contener componentes hijos, directivas, y binding de datos. 


<div>
<h1>{ { titulo }}</h1>
<button (click)="mostrarMensaje()">Mostrar mensaje</button>
</div>

3. Los Estilos (CSS)

Los estilos se aplican a través del archivo componente.component.css. Los estilos son encapsulados de manera predeterminada, lo que significa que solo afectarán al componente en cuestión y no a otros componentes de la aplicación. 


h1 {
  color: blue;
}

button {
  background-color: green;
  color: white;
  padding: 10px;
}

→ Creando un Componente en Angular

Para crear un nuevo componente en Angular, puedes utilizar el comando ng generate component nombre-componente o ng g c nombre-componente en la terminal.


  ng generate component my-component

Este comando generará automáticamente cuatro archivos para el componente:

  • my-component.component.ts: La clase del componente.
  • my-component.component.html: El template del componente.
  • my-component.component.css: Los estilos del componente.
  • my-component.component.spec.ts: El archivo de pruebas para el componente (opcional).

Uso de un Componente

Una vez que hayas creado un componente, puedes usarlo en el template de otros componentes. Para esto, simplemente debes agregar el selector del componente como una etiqueta HTML dentro del template de otro componente. 


  <app-mi-componente></app-mi-componente>

Enlace de Datos (Data Binding)

Uno de los aspectos más poderosos de Angular es el enlace de datos, que permite conectar la lógica de la clase del componente con su vista (HTML). Existen varias formas de enlace de datos en Angular:

  • ▪️Interpolación ({ { }}): Para mostrar valores de la clase en el template.
  • ▪️Property Binding ([ ]): Para enlazar propiedades HTML con propiedades de la clase.
  • ▪️Event Binding ( ( ) ): Para enlazar eventos de la vista con métodos de la clase.
  • ▪️Two-Way Binding ( [( )] ): Para crear una comunicación bidireccional entre la vista y la clase.
