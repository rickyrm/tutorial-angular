Instalación de Angular

→ Detalles de la instalación

¿Se precisa instalación?

Sí, para trabajar con Angular es necesario instalar Angular CLI (Command Line Interface), Node.js y npm (Node Package Manager), que permiten crear, gestionar y ejecutar aplicaciones Angular.

¿Qué pasos hay que realizar para realizar la instalación?

Los pasos necesarios son los siguientes:

  1. Instalar Node.js: Descargue e instale Node.js desde su página oficial.
  2. Instalar Angular CLI: Abra la terminal o línea de comandos y ejecute el comando: 
    npm install -g @angular/cli
  3. Crea un nuevo proyecto: Ejecute el comando 
    ng new nombre-del-proyecto
    para crear el proyecto y configurar la estructura inicial.
  4. Ejecutar el servidor: Acceda al directorio del proyecto con 
    cd nombre-del-proyecto
    y ejecute 
    ng serve
    para iniciar el servidor de desarrollo.
  5. Accede a la aplicación: Abre tu navegador en http://localhost:4200 para ver la aplicación en ejecución.

¿Versiones necesarias?

Es recomendable utilizar las siguientes versiones:

  • Node.js: La versión LTS (Long Term Support), que puede consultarse en nodejs.org.
  • Angular CLI: La última versión estable, que puede instalarse mediante 
    npm install -g @angular/cli
    .

¿Rutas para tener en cuenta?

Las rutas principales a tener en cuenta son:

  • Proyecto raíz: La carpeta que contiene todos los archivos del proyecto Angular, generada con ng new nombre-del-proyecto.
  • src/app: Carpeta donde se encuentran los componentes, módulos y servicios de la aplicación Angular.
  • src/environments: Aquí se configuran las variables de entorno para los entornos de desarrollo y producción.
  • public: Archivos públicos como el archivo index.html que se sirve a través del servidor.

¿Variables de entorno?

Angular utiliza archivos de configuración en src/environments/ para gestionar variables de entorno, como la configuración para los entornos de desarrollo y producción.

En el archivo src/environments/environment.ts puedes definir variables para el entorno de desarrollo, mientras que en src/environments/environment.prod.ts puedes configurar valores específicos para producción.

¿Necesita base de datos?

No, Angular no requiere una base de datos para su funcionamiento. Sin embargo, las aplicaciones Angular suelen interactuar con una base de datos a través de servicios web, como APIs RESTful, para gestionar los datos. El backend puede ser cualquier tecnología que proporcione estas APIs, como Node.js con Express, Java Spring Boot, o incluso servicios de terceros.
