Funcionamiento

→ Explicación del funcionamiento del framework

¿Actúa sobre páginas, ficheros, secciones…?

Angular actúa principalmente sobre componentes, que son las unidades básicas de construcción en el framework. Cada componente controla una vista específica (una parte de la interfaz de usuario) y está asociado a un archivo HTML, un archivo CSS/SCSS y un archivo TypeScript.

Además, Angular también utiliza módulos para agrupar componentes relacionados y servicios para manejar la lógica de negocio y compartir datos entre diferentes partes de la aplicación.

¿Cómo se editan/modifican los ficheros que actúan en el framework?

  1. Editar componentes:

    Los componentes se editan modificando sus archivos correspondientes:

    • .html: Define la estructura y el contenido visual.
    • .ts: Contiene la lógica del componente.
    • .css/.scss: Define el estilo visual.

    Por ejemplo, para modificar el contenido visual de un componente, edita su archivo component.html.

  2. Servicios y lógica compartida:

    Los servicios se encuentran en archivos .ts y se utilizan para encapsular lógica y datos que se comparten entre diferentes componentes. Puedes modificarlos según sea necesario para adaptarlos a las necesidades de tu aplicación.

  3. Otros archivos:

    El archivo angular.json permite configurar opciones globales del proyecto, como rutas, estilos y scripts adicionales.

¿Cómo se referencian los ficheros creados desde otras aplicaciones (si es el caso)?

En Angular, los ficheros no se referencian directamente desde otras aplicaciones como en el caso de librerías tradicionales. Sin embargo, se pueden crear bibliotecas reutilizables dentro de un proyecto Angular que luego pueden compartirse e instalarse en otros proyectos. Esto se hace siguiendo estos pasos:

  1. Crear una biblioteca:

    Usa el comando ng generate library mi-biblioteca para generar una nueva biblioteca.

  2. Compilar la biblioteca:

    Ejecuta ng build mi-biblioteca para compilarla y generar los paquetes necesarios.

  3. Publicar la biblioteca:

    Publica la biblioteca en un registro de paquetes, como npm, o compártela localmente.

  4. Usar la biblioteca en otro proyecto:

    Instala la biblioteca en el nuevo proyecto con npm install mi-biblioteca y luego impórtala en los módulos donde se necesite.
