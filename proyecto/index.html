Estructura de un Proyecto Angular

La estructura de un proyecto Angular está diseñada para ser escalable y organizada. Te ayuda a mantener un código limpio, modular y fácil de mantener. Aquí se explican las carpetas y archivos principales que encontrarás al crear un nuevo proyecto Angular.

→ Estructura de Carpetas y Archivos

Al crear un nuevo proyecto Angular con el comando ng new nombre-del-proyecto, Angular genera una serie de carpetas y archivos predeterminados. Esta es la estructura básica del proyecto:


src/
app/
    app.component.css
    app.component.html
    app.component.ts
    app.module.ts
assets/
    environments/
    index.html
    main.ts
    styles.css
    tsconfig.json

1. Carpeta src/

La carpeta src/ es donde reside el código fuente de la aplicación. Dentro de esta carpeta, encontrarás varios subdirectorios que contienen todo lo necesario para que tu aplicación funcione.

2. Carpeta app/

La carpeta app/ es la más importante de todas, ya que contiene los componentes, servicios y módulos principales de la aplicación.

  • app.component.ts: Este es el componente raíz de la aplicación. Contiene la lógica y el template básico.
  • app.component.html: Es el archivo HTML de la raíz, que generalmente contiene la estructura básica de la página y el <router-outlet>.
  • app.module.ts: Este archivo es el módulo principal de la aplicación. Aquí se definen los componentes, servicios y módulos importados.

3. Carpeta assets/

En esta carpeta se colocan todos los recursos estáticos de la aplicación, como imágenes, fuentes, archivos de datos, etc.

4. Carpeta environments/

La carpeta environments/ contiene los archivos de configuración para diferentes entornos de ejecución, como desarrollo y producción.

  • environment.ts: Configuración para el entorno de desarrollo.
  • environment.prod.ts: Configuración para el entorno de producción.

5. Archivos principales

Estos son algunos de los archivos más importantes que encontrarás en el directorio raíz de tu proyecto:

  • index.html: El archivo HTML principal que se carga cuando accedes a la aplicación. Aquí se carga la aplicación Angular.
  • main.ts: Este archivo arranca la aplicación Angular. Aquí se inicializa el módulo principal (AppModule) y se arranca el proceso de compilación.
  • styles.css: El archivo de estilos globales donde puedes agregar estilos para toda la aplicación.
  • tsconfig.json: Archivo de configuración de TypeScript, donde se definen las opciones del compilador.

→ Modularidad en Angular

Angular está basado en un sistema modular, lo que significa que los componentes, servicios y otros elementos de la aplicación están organizados en módulos. Esto mejora la organización del código y facilita la reutilización.

1. ¿Qué es un Módulo?

Un módulo en Angular es un conjunto de componentes, servicios y otros elementos relacionados. El módulo principal es el AppModule, pero puedes crear módulos adicionales para organizar el código de manera más eficiente. 


@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule],
  providers: [],
  bootstrap: [AppComponent]
  })
export class AppModule { }

2. Cómo organizar módulos

En proyectos grandes, puedes crear módulos específicos para funcionalidades, como un módulo de usuarios, un módulo de productos, etc. Cada uno de estos módulos puede tener su propio conjunto de componentes, directivas y servicios.

3. Modularización con Lazy Loading

Angular permite usar "Lazy Loading" para cargar módulos de manera diferida, lo que ayuda a mejorar el rendimiento de la aplicación cargando solo lo necesario cuando el usuario lo necesita.
